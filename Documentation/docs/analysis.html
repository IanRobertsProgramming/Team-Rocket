Entity Objects

There are two primary entity objects identified using noun-extraction. The project will store information about ‘first generation’ Pokémon and any teams created by the user

  1. Pokemon: the entity object that stores information about each Pokémon. It will allow Pokémon to be added to teams, to enable the user to easily see information about the Pokémon in one place, and to allow comparison between two Pokémon species. All attributes are set when entered into the database and immutable (unless changed at an administrative level inside the database).

    2. Attribute Name Attribute Variable Name Type Description
    Name name String the name of the Pokémon
    HP hp integer the base health statistic*
    Attack attack integer the base attack statistic*
    Defense defense integer the base defense statistic*
    Special Attack special_attack integer the base strength of the Pokémon’s special attack*
    Special Defense special_defense integer the base strength of the Pokémon’s special defense move*
    Speed speed integer the base speed statistic*
    Evolve Level evolve_level integer the level at which the Pokémon evolves into a different Pokémon (the value is NULL if it does not have an evolve level)
    Location(s) location String the location or locations where the Pokémon can be found in the game
    Type type String the Pokémon type statistic (water, fire, grass, rock, etc.)
    Effectiveness effectiveness list of type String an array of all the strengths and weaknesses of a particular Pokémon when fetched from the database.

    *Each Pokémon species has a base level of HP, Attack, Defense, Sp. Attack, Sp. Defense, and Speed regardless of the moves they have and level that they are.


  2. Team: the entity object that stores information about a Pokémon team (of max 6 Pokémon to match the in-game team size) that has been created. It enables the user to retrieve teams that they have created and saved earlier and then to make changes to them. All attributes in the team entity project are set when the user saves the team that has been created.

    3. Attribute Name Attribute Variable Name Type Description
    Team Name team_name string the name of the team, created by the user
    Owner owner User object the user who created the team
    Members members list of Pokemon objects the Pokémon on the team
    Number number integer the number of members out of 6
    *Team Name was removed as our team decided to not support the adding of Team Name.

  3. User: the profile for the user that tracks their information. Is created by the user when they sign up to use the web application. Information is updated by them at their convenience (except their user ID which will not change).

    4. Attribute Name Attribute Variable Name Type Description Creation Period
    Username username String the name of the user Set during account creation
    User ID user_id integer a number to identify the user Set during account creation
    Password password String the user’s password Set during account creation
    Saved Teams saved_Teams list of Team objects the teams that the user has created and saved (up to three) Set by the user during team creation or deletion, subject to change
    Saved Pokemon saved_pokemon list of Pokemon objects Pokémon that the user has chosen to save Set by the user after choosing to save or remove a Pokémon from their Saved Pokémon list, subject to change

Entity Objects Diagram

Boundary Objects

Each boundary option has two sections to it. What details the information that the object must be input by the user and/or the information that will be presented to the user. How is a broad explanation on how the information will be presented and/or the action required from the user to obtain the information necessary.

  1. Log-in:

      2. What: The log-in menu will be used to record the user credentials and to identify the particular user. It will require two pieces of information from the user: the user ID and the password. If the credentials provided are not associated with a recorded user, an error message should pop up and ask for different credentials. Additionally, if the user forgets log-in password, the web page will direct the user to the help centre, where the user can reset the password through texts or emails.

      How: The user will type in the information. The error message and help centre will be a separate window.

  2. Registration:

      3. What: Presents the user with an option to register a new account. When a user attempts to create an account that already exists (based on the email being used), an error message will be displayed to indicate the registration failure. Another error message will be displayed if the information provided does not satisfy the requirements (e.g. username minimum/maximum lengths, etc.).

      How: The user will type in information. The error message(s) will pop up in a separate window.

  3. Home:

      4. What: It will present the user with four options that they can be directed to: the search option, saved Pokémon, saved teams, and comparisons.

      How: The object should have four buttons, corresponding to each of the above options, that the user can click.

  4. Search:

      5. What: The user must input the name of a Pokémon that they want to search. The Pokémon that matches the search will be presented to the user and the user will be prompted to view this Pokémon or search again. However, if the Pokémon is not found, the search will return none and prompt an error message - “There is no matching result”.

      How: The user will type in the information.

  5. Profile:

      6. What: The profile boundary object presents the user with all of the statistics in the Pokémon entity object (Name, HP, Attack, Defense, Special Attack, Special Defense, Speed, Evolve Level, Location(s), Type, and Effectiveness) and a photo of the Pokémon.

      How: A table of the statistics and a photo above it.

  6. Saved Pokémon:

      7. What: A list of the Pokémon that the user has chosen to save or favorite for future quick lookup.

      How: A list of the names of the Pokémon, sorted alphabetically.

  7. Comparison:

      8. What: The user must input which two Pokémon will be compared and then the user is presented with statistics (Name, HP, Attack, Defense, Special Attack, Special Defense, Speed, Evolve Level, Location(s), Type, and Effectiveness) from two Pokémon and their photos respectively. Should the user input two of the same Pokémon, they will receive a warning/error message stating that “The two Pokémon are the same,” and the program will only display one copy of the data.

      How: In a table, similar to the Profile object.

  8. Team Selection Interface:

      9. What: Allows users to create teams and presents the user with a list of teams, up to three, that they have created. Users are not able to create more than three teams and if they try to, an error message will pop up to inform them that three teams at maximum is the rule.

      How: A list of the names of the teams, sorted alphabetically.

  9. Team:

      10. What: Presents the user with the information on a team, including Pokémon in the team. The user can input Pokémon that they would like to place on the team in an empty slot or to replace a team member. However, each team is only allowed to input up to six Pokémon, any member who attempts to input more than 6 Pokémon will be prompted with an error message- “You’ve reached the maximum limit of Pokémon in a team, this Pokémon cannot be added.”

      How: The user will be able to click on slots and choose from a list. The slots will be presented to the user with six slots organized into two rows.

  10. Header:

      11. What: Presents the user with the title of the project (“Pokémon Encylopedia”) and the name of the current webpage. It enables the user to return to the Homepage.

      How: The user can press a button to return to the homepage.

Control Objects

The following control objects were determined based on verb extraction. The following descriptions detail what actions the control objects will take.

Login Account Determine if the credentials entered by the user are correct (i.e. that they match to a saved user). If it is correct, it will allow the user entry to the website.
Search Use the name entered by the user to find the Pokémon that matches it.
Create Team Creates a team for the user by saving the name and Pokémon selected (as determined by the user).
Retrieve Team Retrieves the saved team(s) from the database for viewing and modification by the user.
Display Team Displays a team
Save Pokemon Saves the selected Pokémon into the ‘Saved Pokémon’ list
Retrieve Pokemon Retrieves a Pokémon from the database
Display Pokemon Displays a Pokémon
Compare Display two Pokémon for comparison
